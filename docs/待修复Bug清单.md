## 已验证 ✅

### Bug 1: QMT HTTP 服务端并发（原 Bug 5）

- **来源**：说明文档
- **问题**：盯盘系统通过 HTTP 接口获取个股实时行情（`/quotes`），高峰时段 QMT HTTP Server（192.168.1.33:5000）可能无响应
- **验证结果**：2026-06-01 上午证实现过，服务端完全不响应 HTTP 请求（TCP 握手成功但无数据返回），Watcher 每轮扫描卡 120s 超时。重启 QMT Server 后恢复正常
- **方向**：需加超时降级机制；考虑给采集器和盯盘用不同端口

### Bug 2: QMT /all_quotes amount 字段（原 Bug 6）

- **问题**：板块资金量能计算依赖 `amount` 字段，需在真实 QMT 数据上验证
- **验证结果**：`market_snapshots` 表 99% 行 amount > 0，字段有效。板块量比计算正常

### Bug 3: QMT Collector socket 稳定性（新增）

- **问题**：Watcher 通过 `DataCollectorClient`（TCP socket）接收 Collector 推送的指数和全市场数据。Collector 侧（macOS，`qmt-collect` 进程）的稳定性待验证，包括断连重连、数据断层恢复
- **状态**：2026-06-01 已部署 Collector，持续采集模式（每轮循环拉取），待观察长时间运行的稳定性
