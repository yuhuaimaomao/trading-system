### orchestrator.py 核心改造

```python
class MonitorOrchestrator:
    def run(self):
        """盯盘主循环。只做编排，不含业务逻辑。"""
        while self.clock.is_trading():
            snapshot = self.market_data.fetch()
            regime = self.patterns.classify(snapshot)
            positions = self.risk.check(snapshot, self.account)
            signals = self.signal_engine.evaluate(snapshot, regime)
            for s in signals:
                self.execution.execute(s)
            self.notification.flush()
```

## 十、改造后的强制规范

改造完成后，`quant-system/CLAUDE.md` 永久生效：

```markdown
# CLAUDE.md — 开发规范（违反 = 代码跑不起来）

## 铁律

1. 所有数字进配置。代码里出现 80、0.005、200_000 等 → pre-commit 拦截
2. 所有文件 ≤ 400 行。超过 → pre-commit 拦截
3. shared/ 下禁止 IO。import sqlite3/requests → pre-commit 拦截
4. 所有函数有类型标注。mypy --strict 不过 → pre-commit 拦截
5. 域之间互不 import。monitor/ import review/ → pre-commit 拦截
6. 加功能 = 加文件。不改现有文件
7. 改代码前跑 make check。改完跑 make test + make e2e。0 失败才提交

## 放文件规则

- 不知道放哪？查 docs/2026-06-01-最终架构设计.md 的映射表
- 数据源/IO 通道 → shared/adapters/ 或 shared/collectors/
- 类型/配置/工具 → shared/types/ 或 shared/config/ 或 shared/utils/
- 盯盘实时逻辑 → monitor/
- 复盘盘后逻辑 → review/
- 个股分析 → stock/

## 提交前

make ci    # 跑全部：check + test + e2e + arch-test
           # 任何一步失败 → 不能提交
```
