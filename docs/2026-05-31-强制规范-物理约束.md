# 强制规范系统 — 不让代码变屎山的物理约束

> 架构文档管不了我写代码的手。需要物理约束——不遵守就跑不起来。

---

## 一、三层拦截

```
第1层: pre-commit → 本地提交前拦截 (2秒)
        ↓ 通过
第2层: CI 自动检查 → push 后拦截 (30秒)
        ↓ 通过
第3层: 架构测试 → 测试套件的一部分 (1秒)
        ↓ 全部通过
      合并
```

---

## 二、第1层：pre-commit（本地提交前，无法绕过）

```yaml
# .pre-commit-config.yaml
repos:
  - repo: local
    hooks:
      # 1. 类型检查 — 不通过不能 commit
      - id: mypy
        name: mypy strict
        entry: mypy src/ --strict
        language: system
        types: [python]
        pass_filenames: false

      # 2. 代码风格 — 不通过不能 commit
      - id: ruff
        name: ruff check
        entry: ruff check src/ tests/
        language: system
        types: [python]
        pass_filenames: false

      # 3. 禁止魔法数字 — 不通过不能 commit ★核心★
      - id: no-magic-numbers
        name: 禁止硬编码数字
        entry: python scripts/check_no_magic_numbers.py
        language: system
        pass_filenames: false

      # 4. 文件大小限制 — 不通过不能 commit
      - id: file-size
        name: 单文件不超过400行
        entry: python scripts/check_file_size.py
        language: system
        pass_filenames: false

      # 5. domain 层不依赖 IO — 不通过不能 commit
      - id: domain-purity
        name: domain层零IO依赖
        entry: python scripts/check_domain_purity.py
        language: system
        pass_filenames: false
```

**效果**：写一行 `INITIAL_CAPITAL = 200_000` 在 domain/ 里 → `git commit` 直接报错，commit 不上去。

### 各检查脚本

```python
# scripts/check_no_magic_numbers.py
"""扫描 domain/ 和 application/，禁止出现魔法数字。

允许的数字字面量：
  - 0, 1, -1
  - 100 (股数倍数)
  - 0.0, 0.5, 1.0 (基础比例)
  - 字符串里的数字 (f-string, 日志)

禁止的数字字面量：
  - 80 (一看就是 RSI 阈值)
  - 0.005 (一看就是波动阈值)
  - 200_000 (一看就是金额)
  - 任何 >= 2 的整数（除非是 100 的倍数）
"""

import ast, sys, re
from pathlib import Path

FORBIDDEN_PATTERNS = [
    (r'(?<![.\w])\d{3,}(?!\d)', '多位整数 — 应进配置'),
    (r'(?<![.\w])0\.\d{3,}(?!\d)', '多位小数 — 应进配置'),
]

def check_file(path: Path) -> list[str]:
    errors = []
    tree = ast.parse(path.read_text())
    for node in ast.walk(tree):
        if isinstance(node, ast.Constant) and isinstance(node.value, (int, float)):
            v = node.value
            # 白名单
            if v in (0, 1, -1, 0.0, 0.5, 1.0, -1.0): continue
            if isinstance(v, int) and v % 100 == 0: continue
            if isinstance(v, float) and v in (0.25, 0.75): continue
            # 黑名单
            errors.append(f"{path}:{node.lineno} 魔法数字 {v} — 应进配置")
    return errors

def main():
    all_errors = []
    for d in ["src/domain", "src/application"]:
        for f in Path(d).rglob("*.py"):
            all_errors.extend(check_file(f))
    if all_errors:
        print("\n".join(all_errors))
        print(f"\n共 {len(all_errors)} 个魔法数字。全部进 config/defaults.yaml。")
        sys.exit(1)
    print("✅ 无魔法数字")
```

```python
# scripts/check_file_size.py
"""单文件不超过 400 行"""

import sys
from pathlib import Path

MAX_LINES = 400

def main():
    errors = []
    for f in Path("src").rglob("*.py"):
        lines = len(f.read_text().splitlines())
        if lines > MAX_LINES:
            errors.append(f"{f}: {lines} 行 (上限 {MAX_LINES})")
    if errors:
        print("\n".join(errors))
        print(f"\n共 {len(errors)} 个文件超标。拆分。")
        sys.exit(1)
    print(f"✅ 所有文件 ≤ {MAX_LINES} 行")
```

```python
# scripts/check_domain_purity.py
"""domain/ 目录下禁止 import 任何 IO 库"""

import sys
from pathlib import Path

FORBIDDEN_IMPORTS = [
    "infrastructure", "sqlite3", "requests", "httpx",
    "aiohttp", "telegram", "qmt", "akshare",
    "json",               # 允许，但建议用 msgspec
]

def main():
    errors = []
    for f in Path("src/domain").rglob("*.py"):
        text = f.read_text()
        for imp in FORBIDDEN_IMPORTS:
            if f"import {imp}" in text or f"from {imp}" in text:
                errors.append(f"{f}: 禁止 import {imp} — domain 层不能有 IO 依赖")
    if errors:
        print("\n".join(errors))
        sys.exit(1)
    print("✅ domain 层零 IO 依赖")
```

## 三、第2层：CI 自动检查（push 后，30秒内）

```yaml
# .github/workflows/ci.yml
name: CI
on: [push, pull_request]

jobs:
  # ... 基础检查同 pre-commit ...

  # 6. E2E 全量验证 — 不通过不能合入 ★核心★
  e2e:
    runs-on: ubuntu-latest
    steps:
      - run: E2E_TEST_MODE=1 python tests/e2e/verify.py --day 2 --scans 240
      - run: |
          if grep -q '"failed": 0' e2e_result.json; then
            echo "✅ E2E 全量通过"
          else
            echo "❌ E2E 失败，禁止合入"
            exit 1
          fi

  # 7. 测试覆盖率 — domain 层必须 100%
  coverage:
    steps:
      - run: pytest tests/domain/ --cov=src/domain --cov-fail-under=95
      - run: pytest tests/ --cov=src/ --cov-fail-under=80

  # 8. 新文件必须有对应测试
  test-coverage-new:
    steps:
      - run: |
          # 检查 src/domain/ 下每个 .py 是否在 tests/domain/ 有对应测试
          python scripts/check_test_coverage.py

  # 9. config 变更审计
  config-audit:
    if: contains(github.event.commits.*.modified, 'config/')
    steps:
      - run: |
          echo "⚠️ 配置文件变更，需要人工审核"
          # 打印 diff 供审核
          git diff origin/main -- config/
```

# tests/architecture/test_layers.py

class TestDomainLayerPurity:
    """domain/ 必须零 IO 依赖"""

    def test_no_infrastructure_imports(self):
        for f in Path("src/domain").rglob("*.py"):
            text = f.read_text()
            assert "infrastructure" not in text, \
                f"{f} 不能 import infrastructure"

    def test_no_database_imports(self):
        for f in Path("src/domain").rglob("*.py"):
            text = f.read_text()
            for db_lib in ["sqlite3", "psycopg", "pymongo"]:
                assert db_lib not in text, f"{f} 不能 import {db_lib}"

    def test_no_network_imports(self):
        for f in Path("src/domain").rglob("*.py"):
            text = f.read_text()
            for net_lib in ["requests", "httpx", "aiohttp", "urllib"]:
                assert net_lib not in text, f"{f} 不能 import {net_lib}"


class TestFileSize:
    """单文件不超过 400 行"""

    def test_all_files_under_limit(self):
        for f in Path("src").rglob("*.py"):
            lines = len(f.read_text().splitlines())
            assert lines <= 400, f"{f} 有 {lines} 行，超过 400 行上限"


class TestConfigCoverage:
    """所有魔法数字必须来自配置"""

    def test_domain_no_magic_numbers(self):
        """domain/ 下禁止出现 >=2 的整数或 >=0.01 的小数字面量"""

## 五、Makefile（一条命令跑全部）

```makefile
# Makefile
.PHONY: check test e2e ci

# 提交前检查（2秒）
check:
	mypy src/ --strict
	ruff check src/ tests/
	python scripts/check_no_magic_numbers.py
	python scripts/check_file_size.py
	python scripts/check_domain_purity.py

# 测试（30秒）
test:
	pytest tests/domain/ -q
	pytest tests/application/ -q
	pytest tests/infrastructure/ -q

# E2E（5秒）
e2e:
	E2E_TEST_MODE=1 python tests/e2e/verify.py --day 2 --scans 240

# 架构测试（1秒）
arch-test:
	pytest tests/architecture/ -q

# CI 全流程（等同于 CI 跑的内容）
ci: check test e2e arch-test
	@echo "✅ 全部通过，可以合入"
```

## 六、效果

```
写代码 → git commit
              │
              ├── mypy 报错？ → 修类型
              ├── ruff 报错？ → 修风格
              ├── 魔法数字？  → 进 config/defaults.yaml
              ├── 文件>400行？ → 拆文件
              ├── domain import QMT？ → 根本 commit 不上去
              └── 通过 → push
                          │
                          ├── CI 跑 E2E
                          ├── E2E 全量不通过？ → 禁止合入
                          └── 通过 → 可以合入
```

**我想在 domain 里写 `if rsi > 80:`？** `git commit` 直接报错：`魔法数字 80 — 应进配置`。

**我想在 domain 里 import QMT 直接取行情？** `git commit` 直接报错：`domain 层禁止 import infrastructure`。

**我想把 500 行逻辑塞一个文件？** `git commit` 直接报错：`文件 503 行，超过 400 行上限`。

这个方案的关键不是检查本身多聪明，是**绕不过去**。`git commit` 被 hook 拦了就是拦了，除非 `--no-verify` 强行跳过。而 CI 那层是第二道关，`--no-verify` 跳过的在 CI 照样被拦住。
