# 板块共振/逆势分析 — 设计文档

日期: 2026-06-01

## 需求

盘中盯盘时，能看到哪些板块与大盘指数共振、哪些逆势而涨/跌。

## 设计决策

| # | 决策 | 选项 |
|---|------|------|
| 1 | 展示方式 | A+B：平时嵌在板块热度TOP5里加标签 + 大盘≥0.3%波动时独立推送 |
| 2 | 核心原则 | **同一时间窗口内**比较板块变化方向 vs 大盘变化方向，不用绝对值 |
| 3 | 独立推送窗口 | 近10分钟（波动爆发段的窗口） |
| 4 | 定时TOP5窗口 | 近50轮（自上次报告以来的中周期窗口） |
| 5 | 覆盖范围 | 行业全量 + 概念全量，各取变化幅度前5 |
| 6 | 深水拉升 | 不重复做，现有 `_check_abnormal()` 的板块异动已覆盖 |
| 7 | 领涨/领跌 | 2-3只，共振下行选领跌，共振上行/逆势走强选龙头 |
| 8 | 时间显示 | 用趋势起点时间（如「13:20起走弱」），不用「持续N轮」 |
| 9 | 异动格式 | 同步优化，匹配共振分析的展示风格 |

## 核心判定逻辑

**关键原则：大盘和板块用同一个时间窗口内的变化方向来比较。**

```
不是: 板块当前+2.8% → 板块涨 → 逆势走强 ✗
而是: 板块近10分钟 4%→2.8% ↓ + 大盘近10分钟 0→-0.7% ↓ → 同向 → 共振下行 ✓
```

### 判定流程

```
1. 确定时间窗口
   - 独立推送: 近10分钟（取 _sector_trend_history 尾部 N 轮的首尾差）
   - 定时TOP5: 近50轮（上次报告到现在的变化）

2. 计算窗口内变化
   index_change  = 窗口尾 _index_prices[-1] - 窗口首 _index_prices[-N]
   sector_change = 窗口尾 _sector_trend_history[-1] - 窗口首 _sector_trend_history[-N]

3. 分类（方向阈值 0.1%）
   大盘↑ + 板块↑ → 📈 共振上行
   大盘↓ + 板块↓ → 📉 共振下行
   大盘↓ + 板块↑ → 🔄 逆势走强
   大盘↑ + 板块↓ → ⚠️ 逆势走弱
   大盘或板块变化不明显 → 不展示
```

### 示例验证

```
场景: 大盘 0 → -0.7%, 电子 4% → 2.8%, 军工 1% → 2%

窗口: 近10分钟
  大盘 change = -0.7%  ↓
  电子 change = -1.2%  ↓  → 同向 → 📉 共振下行
  军工 change = +1.0%  ↑  → 反向 → 🔄 逆势走强

结论: 电子虽然全天+2.8%看着漂亮，但在共振分析里它是共振下行——它在跟着大盘跌。
```

## 数据来源

全部来自 Watcher 已有实例变量：

| 数据 | 来源 | 说明 |
|------|------|------|
| 大盘窗口首/尾价 | `_index_prices[-N]` / `[-1]` | 每轮一个价格点 |
| 板块窗口首/尾涨跌幅 | `_sector_trend_history[industry][-N]` / `[-1]` | 每轮行业均值 |
| 概念窗口首/尾涨跌幅 | 同上逻辑，新增 `_concept_trend_history` | 目前 `_concept_stats` 存当前值但没存历史 |
| 涨跌家数 | `_sector_stats[industry].up/down` | 当前快照值 |
| 趋势起点时间 | 新增 `_sector_trend_start[industry]` | 方向翻转时记录 |
| 领涨/领跌股 | `_market_snapshot` 按行业分组排序 changePct | 每轮更新 |
| 量能标签 | `_sector_stats[industry].vol_ratio` | >1.5→放量，<0.5→缩量 |

## 展示格式

### A. 嵌在板块热度 TOP5 中（每50轮，~50分钟窗口）

在现有 `📊 板块热度 TOP5` 消息中，每行加共振/逆势标签，底部加逆势板块独立一节：

```
📊 板块热度 TOP5  14:20
   🥇 电子: +2.31%  📈共振 | 涨18跌3 🔥放量
      龙头: 立讯精密(+5.2%) 歌尔股份(+4.1%)
   🥈 计算机: +1.85%  📈共振 | 涨15跌5
      龙头: 中科曙光(+4.3%) 浪潮信息(+3.9%)
   ...

🔄 逆势走强（大盘↓但板块↑）:
   银行: +0.85%  涨9跌2  10:15起走强
      龙头: 工商银行(+2.3%) 建设银行(+1.9%)
```

### B. 大盘波动 ≥0.3% 时独立推送（~10分钟窗口）

```
🏭 板块共振/逆势分析  14:35  上证: 3,124.56 -0.70%

   ━━━ 📉 共振下行 ━━━
   电子      -1.20%  涨3跌22  13:20起走弱  量缩
    领跌: 立讯精密(-5.8%) 歌尔股份(-5.1%) 海康威视(-4.3%)
   计算机    -0.85%  涨4跌18  13:35起走弱
    领跌: 浪潮信息(-4.9%) 中科曙光(-4.2%)

   ━━━ 🔄 逆势走强 ━━━
   军工      +1.00%  涨15跌5  13:25起走强  🔥放量
    龙头: 中航沈飞(+6.1%) 航发动力(+4.5%) 中航西飞(+3.7%)
   黄金概念  +0.65%  涨8跌3   13:40起走强
    龙头: 山东黄金(+3.5%) 中金黄金(+2.9%)
```

注意：板块涨跌幅显示的是**窗口内变化**（如电子 -1.20%），不是全天绝对值。

### C. 异动检测格式优化

```
🔥 板块异动 电子  14:35
   立讯精密(+5.2%) 歌尔股份(+4.1%) 京东方A(+3.8%)

🏭 急速拉升  14:35
   立讯精密 +3.5%  歌尔股份 +2.8%  京东方A +2.1%  等12只

📊 量比暴增  14:35
   歌尔股份 5×  立讯精密 4×  京东方A 3×

🚀 逼近涨停  14:35
   中航沈飞(+7.2%) 航发动力(+7.0%)
```

## 两个触发的区别总结

| | 独立推送 | 定时TOP5 |
|------|------|------|
| 触发条件 | 大盘近10分钟波动 ≥0.3% | 每50轮固定触发 |
| 时间窗口 | 近10分钟 | 近50轮（~50分钟） |
| 目的 | 响应突发波动，看谁跟风谁抵抗 | 中周期趋势报告 |
| 板块变化显示 | 窗口内变化（如 -1.20%） | 窗口内变化 |
| 去重 | ≥15轮间隔 + 方向/排名无明显变化不重复 | 固定频率，无需去重 |

## 实现计划

### 文件变更

1. **新增** `trade/monitor/sector_resonance.py` — 共振/逆势分析核心逻辑
   - `class SectorResonanceAnalyzer`
   - `analyze(window_start, window_end, index_prices, sector_trend_histories, concept_trend_histories)` → 分类结果
   - `get_leader_stocks(sector_name, snapshot, cache)` → 领涨/领跌股
   - `format_resonance_message(result)` → 独立推送格式化
   - `format_top5_label(result)` → TOP5标签格式化

2. **修改** `trade/monitor/sector_context.py`
   - 新增 `_concept_trend_history`: dict[str, list[float]] — 概念板块趋势历史
   - 新增 `_sector_trend_start`: dict[str, datetime] — 方向翻转时记录时间
   - `_update_sector_trends()` 中同步维护 `_concept_trend_history` 和 `_sector_trend_start`

3. **修改** `trade/monitor/abnormal.py`
   - `AbnormalDetector.detect_sector()` 输出格式对齐新风格

4. **修改** `trade/monitor/watcher.py`
   - `_scan()` 中新增独立推送触发逻辑
   - `_check_sector_heat()` 集成共振标签

5. **修改** `trade/monitor/sector_heat.py`
   - `SectorHeatMonitor.check()` 接受外部传入的共振分类结果，注入标签

### 数据流

```
每3轮 _update_sector_trends()
  ├─ 维护 _sector_trend_history (已有)
  ├─ 维护 _concept_trend_history (新增)
  ├─ 维护 _sector_trend_start (新增，方向翻转时记录)
  └─ _index_prices 由 collector 推送不断追加

每轮 _scan():
  └─ 检测大盘近10分钟波动 ≥0.3% → _push_resonance_alert()
      ├─ 取窗口: 近10轮的数据 (index_prices + trend_histories)
      ├─ 计算每个板块窗口内变化
      ├─ 与大盘窗口内变化对比 → 四分类
      ├─ 各取前5
      └─ 格式化推送

每50轮 _check_sector_heat():
  └─ SectorHeatMonitor.check() 输出消息
      └─ 同时算近50轮窗口的共振分类 → 注入每行标签
```
