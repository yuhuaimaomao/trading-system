# 尾盘选股次日卖策略设计

> 2026-06-10 初稿，待讨论验证后落地

## 策略定位

赚**隔夜信息差 + 次日惯性冲高**的钱。尾盘全天走势明朗、不确定性最低，次日早盘惯性冲高后离场，不承担日内波动风险。

A股 T+1 限制正好被这个策略利用——尾盘买入只需承担隔夜到次日开盘的风险，全天随机波动跟你无关。

## 选股条件（四个维度同时满足）

### 1. 量价结构（权重最高）

```
当日涨幅       2% ~ 7%（不追涨停，不吃跌停）
尾盘30分钟量    > 全天均量的 40%（14:30~15:00）
尾盘价格        > 全天 VWAP（真金白银买，不是诱多）
收盘价位置     当日最高价的 2% 以内（拒绝尾盘跳水）
日线振幅       3% ~ 8%（有活力但不至于分歧过大）
```

**核心信号**：尾盘放量上翘。全天平平无奇的票，尾盘突然放量拉升，大概率有资金抢筹，次日早盘有惯性。

### 2. 趋势过滤

```
均线    5日 > 10日 > 20日（多头排列）
收盘    站上 5 日线
近3日   没有放量大阴线（-3%以上且放量）
排除    连续涨停后首日开板（高位分歧风险大）
```

只做上升趋势中的尾盘买点。下降趋势博反弹是送钱。

### 3. 资金面

```
主力净流入   > 0（大单净买）
换手率       3% ~ 15%（太冷没关注，太热是出货）
流通市值     30亿 ~ 500亿（太小被操控，太大拉不动）
排除        当日跌停又打开的票
```

### 4. 板块确认

```
板块涨幅     当日排名前 20
板块涨停家数  >= 2（有板块效应）
排除        板块内唯一涨的（孤狼容易一日游）
```

## 买入执行

```
时间窗口   14:50 ~ 14:57（最后3分钟集合竞价不可撤单，避开）
仓位       单票 ≤ 20% 总资金，同时持仓 ≤ 3只
下单方式   市价买入，不挂限价（尾盘时间宝贵）
```

**注意**：14:57~15:00 是收盘集合竞价，挂单后不能撤。14:57 前必须完成操作。

## 次日卖出策略

### 情况A：高开（开盘价 > 昨收 +1%）

```
9:25  集合竞价出 1/2（锁定利润）
剩余  设止损在昨收价，持有到 10:00
      10:00 前涨幅 > 3% → 全出
      10:00 回落但还在水上 → 也出
```

尾盘股次日高开 = 惯性兑现，大部分冲一下就回落，不要贪。

### 情况B：平开（开盘价在昨收 ±1% 以内）

```
等 9:30~9:45 第一次冲高，挂昨收 +2% 卖单
10:00 没到目标 → 市价出
9:45 前跌破昨收 -1.5% → 直接止损
```

### 情况C：低开（开盘价 < 昨收 -1%）

```
等 9:30~9:35 是否有快速修复
5分钟内没回到昨收以上 → 市价全部止损
快速拉回 → 按平开策略处理
```

### 通用铁律

- **14:00 前必须清仓**，尾盘再涨也不留
- 单票亏损超过 2% 无条件止损
- 高开低走跌破开盘价就卖，不等

## 胜率预期

```
目标胜率      55% ~ 65%
盈亏比        1.2 ~ 1.8（赚 2% 走，亏 1.5% 割）
单次期望      约 +0.3% ~ +0.5%
月度期望      做对 20 笔 → 约 6% ~ 10%（不含复利）
```

核心竞争力不在胜率，在**严格止损**。大多数人亏钱是因为止损不果断——低开心想"再等等"，越等越低。

## 不做的情况

```
大盘当日跌幅 > 1.5%   系统性风险
周五                   隔周末不确定性 + 周一低开概率高
长假前最后2个交易日    节后跳空风险
选不出符合条件的票     空仓，不将就
当日有重大政策/消息冲击 不确定性过高
```

## 系统架构定位

### 盯盘管线引擎3

尾盘选股是盯盘管线（`trade/`）的**第三个引擎**，与引擎1、引擎2平级，共用 watcher 主循环：

```
watcher 主循环 (每分钟 tick)
├── 引擎1: 信号跟踪    _check_buy_candidates → evaluate_buy    全时段
├── 引擎2: 盘中发现    _scout_intraday                        全时段
└── 引擎3: 尾盘选股    _scan_tail_close → _execute_tail_close  仅14:30~15:00激活
```

**关键约束**：
- 引擎3 有自己的选股逻辑，不受引擎1/2 的盘面条件约束
- 但三个引擎**共享仓位管理**——引擎3 下单前必须通过同一仓位检查，总仓位、单票上限统一管控
- 次日卖出也跑在 watcher 里（早盘阶段），复用现有风控和消息推送

### 执行时间线

```
14:30  引擎3激活，全市场初筛（量价 + 趋势）
14:45  精选池确认（资金面 + 板块二次过滤）
14:50  AI/人工最终确认（可选）
14:55  自动下单 → 仓位检查 → 执行
14:57  必须完成（收盘集合竞价不可撤单）
次日
  9:25  watcher 检测尾盘持仓 → 集合竞价卖出一半
  10:00 或 14:00 清仓剩余
```

### 需要的模块（均在 trade/ 下）

| 模块 | 说明 |
|------|------|
| `trade/detect/tail_close_scanner.py` | 尾盘筛选器，四个维度过滤 |
| `trade/exec/tail_close_executor.py` | 尾盘下单 + 仓位检查 |
| `trade/decision/tail_close_exit.py` | 次日卖出决策（高开/平开/低开分支） |

### 与现有模块的关系

共用（不新建）：
- 行情数据 — QMT 推送 + Collector，用现有 `_market_snapshot`
- 仓位管理 — 现有 PortfolioRepo，引擎3 不单独维护仓位
- 风控 — 现有 `trade/risk/`，止损/上限统一管控
- AI 服务 — 现有 `system/ai/`
- Telegram — 现有 `system/message/`
- 审计 — 现有 `audit/` 框架，引擎3 决策也入 decision_log

不复用引擎1/2 的选股逻辑（信号跟踪和盘中发现跟尾盘逻辑完全不同），但复用 watcher 主循环和仓位/风控基础设施。
