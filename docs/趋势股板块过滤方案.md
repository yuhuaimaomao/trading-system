# 趋势股板块趋势过滤

## 问题

现有趋势股筛选只看个股 K 线形态（MA 多头排列、斜率、量比等），不检查所属板块是否在走趋势。容易选出「个股形态好但板块没资金关照」的票，被板块拖累。

## 方案

在 `StockReader.get_trend_stocks()` 输出前加一道板块过滤：查趋势票所属概念板块，判断板块是否也在走健康趋势。不满足的个股过滤掉。

## 过滤规则

### 1. 热度过滤（淘汰冷门板块）

查 `sector_hot_history`，最近 3 个交易日至少出现在热点概念板块 **TOP5** 中 ≥1 次。一次都没进过 → 冷门 → 过滤。

### 2. 共振过滤（淘汰弱于大盘的板块）

取近 5 个交易日概念板块每日涨跌幅 vs 上证指数涨跌幅：

- 大盘涨的日子 → 板块涨幅 > 大盘涨幅
- 大盘跌的日子 → 板块跌幅 < 大盘跌幅，或者收红

累计不达标天数 ≥ 阈值（默认 3 天）→ 不同步 → 过滤。

大盘指数数据从 `index_realtime_data` 获取（sh000001 上证指数）。

## 个股归属板块

一只个股可能属于多个概念板块。取 `sector_concept` 表中该股所属板块涨幅最强的 **3 个**，只要有一个通过上述两道过滤即保留。

## 兜底机制

- 如果过滤后候选 < 5 只：放宽至 TOP10
- 如果指数数据缺失（采集断档）：跳过共振检查，仅做热度过滤
- 如果 `sector_hot_history` 无足够数据（新交易日）：仅做共振检查
