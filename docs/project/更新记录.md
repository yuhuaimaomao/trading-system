## 2026-06-08 — 数据层按业务线重构

**变更**：`data/` 目录从「按操作类型」（repo/readers）重构为「按业务线」组织。

**新结构**：
```
data/
├── _base.py          # 共享基类 + connect() + get_db_conn()
├── schema.py         # DDL + 迁移
├── market/           # 行情基础数据（跨域共享）
├── trade/            # 交易线
├── strategy/         # 策略线
├── review/           # 复盘线
├── audit/            # 审计线
├── collect/          # 采集（不动）
├── process/          # 加工（不动）
├── repo/             # [兼容层]
└── readers/          # [兼容层]
```

**关联改动**：
- 业务代码 22 个文件消除 `import sqlite3`，统一通过 `data._base.connect()` 获取连接
- 内联 SQL 约 120 条提取到对应 data/ 模块
- 新增 data 模块：`market/stock_basic.py`, `market/sector_data.py`, `market/events_data.py`, `trade/signals.py`, `trade/orders.py`, `trade/portfolio.py`, `strategy/funnel.py`, `strategy/morning.py`, `strategy/screening.py`, `review/tracker.py`, `review/predictions.py`, `review/analysis.py`, `audit/decision_log.py`
- `data/repo/` 和 `data/readers/` 保留为薄包装兼容层
