## 2026-06-10 — 盯盘管线全面优化 + 五个新因子

### 架构修复

**QMT 行情数据流切换**：`_get_realtime_prices` 从 HTTP 直连 QMT 改为优先使用 Collector TCP 推送的 `_market_snapshot`。Collector 每 7 秒推送全市场 4717 只股票快照到 Watcher 内存，不再每轮扫描走 HTTP（READ_TIMEOUT=120s，曾导致 Watcher 卡死）。

**健康检查修复**：`_cross_validate_index_high_low` 中 `index_close_low` 初始化前跳过校验，避免用 K 线高低点数据与收盘价序列对比产生误报。

### 引擎1 买入过滤放宽

`evaluate_buy()` 十维过滤中大部分拒绝阈值降级为警告+减仓：
- 板块拒绝 -1%→-2%，数据不足从不买到减仓
- 布林%B 拒绝 90→95/98，日内RSI 85→92，MACD bar -0.5→-0.8，KDJ J 100→110
- 买入区 zone_pos 拒绝 0.85→0.95，涨停距离 2%→1%，主力流出 -5%→-8%
- 止跌判断 reversing 从 >1.005x 降到 >0.998x
- 盘前信号跳过 entry_rule 二次拦截
- 复盘/板块候选 AI 异步结果放开推送

**市场宽度分级**：`_check_buy_candidates` 中宽度过滤仅对盘前信号/复盘推荐生效（`source in ("signal","review")`），盘中动态发现和回踩扫描不拦。

### 引擎2 策略转型

从「底部发力」（精确抄底）转为「趋势确认追入」：
- 运行时段 10:00-11:30 → **9:30-14:30**
- 筛选条件 四条件 AND → 两核心（涨幅+板块走强）+ 加分项（量能/加速/板块排名）
- AI 不出售买入：高分(≥70)直接买，中分(50-69)AI辅助，低分仅提醒
- 仓位从固定 5000 改为引擎1统一计算函数
- 板块排名从硬过滤改为加权加分

### 新增五个因子

| 因子 | 实现位置 |
|------|---------|
| 资金流连续趋势 | `StockReader.get_money_flow_trend` → `evaluate_buy` |
| 波动率异动 | `StockReader.get_volatility_breakout` → `evaluate_buy` |
| 板块连续性 | `_scout_layer2_rank` 中使用 `_sector_trend_continuity` |
| 盘口变化率 | `_get_order_book_imbalance` 返回 delta → `evaluate_buy` |
| 开盘强度分析 | `_get_opening_strength` → `evaluate_buy` |

### 关联文件

- `trade/core/watcher.py`: _get_realtime_prices 改用 snapshot, _alert 板块分方向
- `trade/decision/buy.py`: evaluate_buy 门槛放宽 + 五个新因子接入
- `trade/decision/buy_decision.py`: entry_rule 跳过, BuyEvalInput 扩展, 宽度分级
- `trade/detect/intraday_scout.py`: 策略转型, 时段扩展, 板块连续性
- `trade/core/health_checks.py`: index_close 初始化防护
- `data/market/stock_basic.py`: get_money_flow_trend, get_volatility_breakout
- `data/repo/__init__.py`: 新增代理方法

## 2026-06-08 — 数据层按业务线重构

**变更**：`data/` 目录从「按操作类型」（repo/readers）重构为「按业务线」组织。

**新结构**：
```
data/
├── _base.py          # 共享基类 + connect() + get_db_conn()
├── schema.py         # DDL + 迁移
├── market/           # 行情基础数据（跨域共享）
├── trade/            # 交易线
├── strategy/         # 策略线
├── review/           # 复盘线
├── audit/            # 审计线
├── collect/          # 采集（不动）
├── process/          # 加工（不动）
├── repo/             # [兼容层]
└── readers/          # [兼容层]
```

**关联改动**：
- 业务代码 22 个文件消除 `import sqlite3`，统一通过 `data._base.connect()` 获取连接
- 内联 SQL 约 120 条提取到对应 data/ 模块
- 新增 data 模块：`market/stock_basic.py`, `market/sector_data.py`, `market/events_data.py`, `trade/signals.py`, `trade/orders.py`, `trade/portfolio.py`, `strategy/funnel.py`, `strategy/morning.py`, `strategy/screening.py`, `review/tracker.py`, `review/predictions.py`, `review/analysis.py`, `audit/decision_log.py`
- `data/repo/` 和 `data/readers/` 保留为薄包装兼容层
