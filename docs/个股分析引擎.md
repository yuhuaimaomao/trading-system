## 零、核心定位修正

**之前的定位**：多维度分析 → 结构化结论 + 风险标签。不做交易决策。

**修正后**：分析的目的就是辅助决策。如果只输出各维度的独立结论不合成，那是"筛选"不是"分析"。LLM 聚合层（deep 模式）的职责是综合多维度信号 → 输出行动方向判断。

- **quick 模式**：纯规则，盘中用，不烧 token
- **deep 模式**：全维度分析器 → 硬规则一票否决 → LLM 聚合 → 决策仪表盘

## 二、数据流

```
symbol → N 个分析器并行 → N 个 AnalysisResult
                              │
                    ┌─────────┴──────────┐
                    │  hard guardrails   │  ← 规则一票否决（不走 LLM）
                    │  监管≥3⭐ 等        │
                    └─────────┬──────────┘
                              │ 通过
                              ▼
                    ┌──────────────────┐
                    │  LLM Dashboard   │  ← 仅 deep 模式
                    │  综合 N 维信号    │
                    │  交叉判断/权衡    │
                    └──────────────────┘
                              │
                              ▼
                    StockAnalysisReport
                    ├── results: N 维原始结果
                    ├── checklist: TradeChecklist（通过项/风险项）
                    ├── veto_triggered: bool
                    └── llm_dashboard: LLMDashboard | None（quick 模式为 None）
```

## 三、quick vs deep 模式

| 维度 | quick | deep |
|------|-------|------|
| 维度数 | 3-5 个（技术+资金+板块+基本面+监管） | 全部 12 个 |
| LLM | ❌ 不调 | ✅ 聚合输出仪表盘 |
| 延迟 | <100ms | 取决于 LLM 响应（通常 3-10s） |
| 用途 | 盘中 Watcher 快速扫 / 盘前初筛 | 盘后深度研究 / TG `/stock --deep` / AI prompt 注入 |
| 输出 | N 维结论 + 风险汇总 | N 维结论 + 决策仪表盘 |
| **数据来源** | QMT 实时 + 上一日静态 | 盘后东财全量 + 历史序列 |

quick 不走 LLM 的原因：
1. 盘中延迟敏感，3-10s 等不起
2. Watcher 需要确定性的规则判断，不需要权衡
3. 省 token

### 3.1 数据来源场景分类（2026-06-08 讨论确认）

两种数据来源，可用字段完全不同：

| 数据类别 | 盘中 QMT 实时 (`/all_quotes`) | 盘后东财静态 (`stock_basic`) |
|---------|------------------------------|------------------------------|
| 价格/涨跌幅/量比/换手 | ✅ 有 | ✅ 有 |
| 主力净流入 | ✅ 有（amount 字段待验证） | ✅ 有 main_force_net |
| 大单/超大单/中单/小单拆解 | ❓ QMT 是否提供待验证 | ✅ 有 super_large_net 等四档 |
| PE/PB/revenue_growth/profit_growth | ❌ QMT 不提供 | ✅ 有 |
| industry/concepts | ❌ QMT 不提供 | ✅ 有 |
| 均线 MA5/MA10/MA20 | ❌ QMT 不提供 | ✅ 有 |
| MACD/RSI/KDJ/布林 | ❌ QMT 不提供 | ✅ stock_indicators 有 |
| 板块行情 | ❌ QMT 不提供 | ✅ sector_industry/concept 有 |
| 电报/公告/龙虎榜/监管/股东 | ❌ QMT 不提供 | ✅ 各表有 |

**约束**：quick 模式（盘中）只能用 QMT 实时快照，能跑的维度非常有限。deep 模式（盘后）才能跑全维度分析。

```python
class TradeChecklist:
    pass_indicators: list[str]   # 满足的买入/持股条件
```

## 五、策略视角体系（借鉴 daily_stock_analysis 15 策略）

开源项目的 15 个策略不是 15 个独立分析器，而是 **15 种给 AI 的分析视角模板**（YAML 描述 + 工具引用）。我们的 12 个分析器是"数据维度"，策略视角是 LLM 聚合时的"推理框架"。

### 5.1 策略视角 → 分析器映射

| 策略视角 | 对应分析器 | 触发条件 |
|---------|-----------|---------|
| **均线金叉** — 短周期上穿长周期 | technical（升级：金叉/死叉检测） | 检测到近期金叉 |
| **多头趋势** — MA5>MA10>MA20，斜率向上 | technical（已有） | 均线多头排列 |
| **放量突破** — 价格突破阻力 + 量>5日均量 1.5x | vol_price_flow 🆕 | 突破关键位置+放量 |
| **缩量回踩** — 回踩均线支撑 + 量缩至 5 日均量 0.7x 以下 | vol_price_flow 🆕 | 价格回踩支撑+缩量 |
| **底部放量** — 低位区域 + 突然放量 | vol_price_flow 🆕 | 价格低位+量能异动 |
| **箱体震荡** — 在固定区间内运行，碰支撑=买点 | technical（升级：支撑阻力） | 价格在箱体内 |
| **龙抬头** — 回调后强势突破，K 线形态 | technical（升级：形态识别） | 特定 K 线组合 |
| **一阳三阴** — 连跌 3 天后一根放量阳线 | technical（升级：形态识别） | 前三阴+今阳放量 |
| **缠论** — 分型/笔/线段/中枢/买卖点 | technical（升级：缠论分型） | 中枢附近买卖点 |
| **波浪理论** — 1-5 推动浪 + ABC 修正浪 | technical（升级：浪型识别） | 识别当前浪位 |
| **热点主题** — 板块共振+资金集中流入 | sector_attr（升级：板块强度） | 所属板块排名靠前 |
| **事件驱动** — 重大公告/政策催化 | news_sentiment + announcement | 高重要性电报/公告 |
| **情绪周期** — 恐慌/贪婪阶段判断 | news_sentiment（升级：情绪聚合） | 电报密集期 |
| **成长质量** — 营收/利润增速+趋势配合 | fundamental 🆕 + technical | PE 合理+营收增长 |
| **预期重定价** — 基本面预期发生变化 | fundamental 🆕 + LLM 聚合 | 利润增速拐点+股价反应 |

### 5.2 关键设计：分析器输出策略标签

分析器不只输出结论和风险，还要输出**命中的策略标签**，供 LLM 聚合时交叉推理：

```python
# technical analyzer 的 data 字段示例
data = {
    "indicators": {...},
    "strategy_tags": ["多头趋势", "均线金叉", "缩量回踩"],
    "chan_points": {"type": "三类买点", "confidence": "中", ...}  # 如有缠论信号
}
```

LLM 看到的不只是"MA5>MA10>MA20"，而是"这只票从多头趋势 + 均线金叉 + 缩量回踩 三个视角看都有支撑，但筹码面获利盘 95% 偏高"。多个策略视角的交叉点才是决策关键。


## 六、LLM 聚合层设计

### 6.1 输入

把 N 个 `AnalysisResult` 按维度分块，**同时携带策略标签**：

```
【技术面】✅ 多头排列(MA5>MA10>MA20)，MACD 日线多头，RSI6 78 偏高未超买
  策略标签：多头趋势，均线金叉(3天前)，缩量回踩(MA20支撑)
  缠论：未检测到明确买卖点

【资金面】✅ 主力净流入 3200 万（占比 3.2%），超大单和大单同步流入
【量价面】✅ 今日放量突破 MA20 阻力，量比 1.8 > 5 日均量
  策略标签：放量突破

【筹码面】获利盘 72%，90% 筹码集中度 8.5%，单峰密集，价格在峰上方
【监管风控】无监管函件
...
```

### 6.2 硬规则兜底（先于 LLM 执行）

以下情况直接标记 `veto_triggered=True`，不调 LLM：
- 监管风控 ≥⭐⭐⭐
- 连续跌停
- 停牌
- （后续按需扩展）

触发一票否决时，`llm_dashboard.action` 强制为 `WATCH` 或 `SELL`。

### 6.3 LLM 的核心任务：策略视角的交叉推理

不是简单的"利好多=买，利空多=卖"，而是要求 LLM 从策略视角做交叉判断：

```
指令示例：
你是一个量化策略师。请从以下策略视角审视这只股票的数据：
1. 是否满足"多头趋势+放量突破"组合？（趋势延续型买点）
2. 是否有"缩量回踩+均线支撑"信号？（回调确认型买点）
3. 筹码面是否支持当前策略？（获利盘过高=趋势末端风险）
4. 资金面和量价面是否一致？（放量+主力流入=真实突破；放量+主力流出=诱多）

请综合判断并输出 JSON。
```

### 6.4 LLM 输出

要求严格 JSON，字段对齐 `EntryExitPlan`：

```json
{
  "action": "BUY",
  "confidence": "中",
  "dominant_strategy": "多头趋势+放量突破",
  "entry_zone": "15.20-15.50",
  "stop_loss": 14.80,
  "target_price": 18.50,
  "logic_summary": "均线多头排列且放量突破MA20阻力，筹码单峰密集支撑强，资金面机构同步流入。从放量突破视角看买点有效。风险点：RSI偏高+获利盘72%偏高，注意趋势末端回调。",
  "key_risk": "获利盘72%+RSI6=78，短期获利回吐压力"
}
```

### 6.5 Prompt 管理

放在 `system/ai/prompts/stock_analysis.py`，遵循项目规范。

### 7.2 资金面 MoneyFlowAnalyzer ✅ → 升级

**现有**：主力净流入/占比、超大单 vs 大单方向、MA5 斜率、市值规模。

**2026-06-08 讨论要点**：
1. **大单拆解可能消失**：目前 stock_basic 有 super_large/large/medium/small 四档资金拆解，但如果 QMT 后续不再提供拆解，只能看主力净流入总量。分析器需要做降级逻辑。
2. **不能只看当天**：需要看 5 日/10 日/20 日资金趋势（连续流入天数、累计净额、MA5 斜率方向）。当前 `get_money_flow()` 只取最新一天，需要改成取多日序列做趋势判断。

**升级方向**：

| 新增检查项 | 逻辑 |
|-----------|------|
| 资金连续流向 | 近 5/10/20 日主力净流入连续方向，连续流入天数 |
| 累计净额 | 近 5/10/20 日主力净流入累计（万元/亿） |
| 资金加速度 | 近 5 日日均 vs 近 20 日日均，加速流入/流入放缓/转为流出 |
| 大单拆解（如有） | 超大单+大单 vs 中单+小单方向对比，判断机构 vs 散户行为 |
| 降级模式 | 如 QMT 只有净额无拆解，跳过大小单对比，仅做总量+趋势判断

### 7.3 板块归因 SectorAttrAnalyzer ✅ → 升级

**现有**：行业+概念标签，静态文本。

**2026-06-08 讨论要点**：
1. **不能纯静态排名**：板块归因不只是"排名第 3/56"，还需要和市场当前的**叙事方向/热点概念**挂钩。比如最近"物理 AI"火了，要能识别这只票的板块是否属于物理 AI 方向。
2. **deep 模式不需要竞价数据**：竞价是当天开盘强弱信号，deep 模式是盘后分析，看当天竞价没意义。竞价板块共振只对 quick 模式有价值。
3. 需要**市场热点方向数据源**（新闻聚合、概念热度榜、社交讨论趋势），目前没有。

**升级方向**：

| 新增检查项 | 对应策略 | 实现方式 | quick | deep |
|-----------|---------|---------|-------|------|
| 板块实时强度/排名 | 热点主题 | 调 SectorReader 查当日涨跌排名 | ✅ | ✅ |
| 板块资金流方向 | 热点主题 | 板块 main_force_net/ratio | ✅ | ✅ |
| 板块 MA5 趋势 | 热点主题 | 板块连续走强/走弱天数 | ✅ | ✅ |
| 竞价板块共振 | 热点主题 | 所属板块竞价强度排名 | ✅（QMT 实时） | ❌ 不需要 |
| 市场叙事关联 | 热点主题 | 当前市场热门概念 vs 个股所属板块映射 | ❌ 需要额外数据源 | ✅ 需要额外数据源 |

**市场叙事关联数据需求**（待解决）：
- 当前市场在炒什么方向（概念热度榜、新闻聚合关键词）
- 个股板块 → 热点方向的映射关系（如 "AI 算力" → 哪些板块属于这个方向）
- 可能需要：东财/同花顺概念热度榜、雪球热议话题、新闻聚类

**输出示例**（升级后）：
```
✅ 白酒板块今日+2.3%，行业排名 3/56，MA5 趋势向上
   所属概念：超级品牌（+1.8%）、酿酒（+2.1%）、茅指数
   策略标签：热点主题（行业排名前5）
   市场叙事：食品饮料/白酒方向，今日市场关注度中
```

### 7.4 基本面 FundamentalAnalyzer 🆕

**数据源**：`stock_basic`（pe_ttm/pb_ratio/revenue_growth/profit_growth/circ_market_cap）— `StockReader.get_stock_basic()` ✅ 已有。

**策略视角**：成长质量、预期重定价（基本面部分）。

**2026-06-08 讨论要点**：基本面不需要讨论做不做，而是**怎么做更好**。

**基础检查项**（当前数据可做）：

| 检查项 | 逻辑 |
|--------|------|
| PE(TTM) | <0 亏损告警 / 0-15 低估 / 15-30 合理 / 30-60 偏高 / >60 高估 |
| PB | <1 破净 / 1-3 合理 / >5 高估 |
| 营收增速 | >20% 高成长 / 0-20% 稳健 / <0 萎缩 |
| 利润增速 | >30% 高成长 / 营收增但利润降=成本压力 / 双降=经营恶化 |
| 市值规模 | >1000 亿大盘 / 100-1000 中盘 / <100 小盘 |

**进阶检查项**（需要额外数据）：

| 检查项 | 逻辑 | 数据状态 |
|--------|------|---------|
| PE 历史分位 | 当前 PE 在近 1 年的分位数 | ❌ 需要 PE 历史序列（存历史快照或拉 akshare） |
| 增速趋势 | 连续 2-4 季度增速方向变化（加速/减速/拐点） | ❌ stock_basic 只有最新一期，需要历史快照 |
| 盈利质量 | ROE / 毛利率 / 净利率 | ❌ stock_basic 无此字段，需要 akshare 财报接口 |
| 一致预期差 | 分析师预期 vs 实际业绩 | ❌ 需要分析师一致预期数据源 |

### 7.6 量价形态 VolPriceAnalyzer 🆕

**数据源**：`stock_basic`（volume_ratio/amplitude/turnover_rate/avg_vol_5d/avg_vol_20d/OHLC/price/ma5/ma10/ma20/ma60）。

⚠️ 竞价量比（开盘成交额 vs 5 日均值）和盘中 Tick 流需要 QMT 实时数据采集，目前没有。

**策略视角**：放量突破、缩量回踩、底部放量。

**2026-06-08 讨论要点**：不能只看当天，需要多日量价视角。

**当日信号**（现有数据可做）：

| 检查项 | 逻辑 | 对应策略 |
|--------|------|---------|
| 放量突破 | 价格突破 MA20/MA60/布林上轨 且 量比 > 1.5 | 放量突破 |
| 缩量回踩 | 价格回踩 MA10/MA20/布林中轨 且 量比 < 0.7 | 缩量回踩 |
| 底部放量 | 价格在 20 日低位 10% 内 且 量比 > 2 | 底部放量 |
| 量价背离（放量滞涨） | 量比 > 1.5 但涨跌幅 < ±0.5% | ⚠️ 风险信号（主力出货） |
| 量价背离（缩量加速） | 量比 < 0.7 但涨跌幅 > 3% | 筹码锁定良好（可正面） |

**多日信号**（需要取多日量价序列）：

| 检查项 | 逻辑 |
|--------|------|
| 连续放量 | 量比连续 N 天 >1.2，持续有资金关注 |
| 缩量回调后放量 | 前 3 天缩量下跌 + 今天放量上涨 = 洗盘后拉升 |
| 量堆识别 | 某一价格区间持续放量（5-10 天），形成筹码密集区 |
| 量价背离持续 | 连续涨但量持续缩 = 上涨乏力；连续跌但量缩 = 抛压减轻 |

**待采集**（目前没有）：
- 竞价异动（竞价量比 > 3）— 需要 QMT 竞价数据采集

`data.strategy_tags`：命中放量突破 → `["放量突破"]`；命中缩量回踩 → `["缩量回踩"]`；量价背离 → 不进策略标签，进 `risk_flags`。

## 十、设计原则（不变）

1. **分析器只做判断**：调 Reader 拿数据 → 分析 → 返回结论和风险标签
2. **Reader 只做查询**：返回原始 dict/list，不做格式化、不做判断
3. **data vs conclusions 分离**：`data` 给程序消费，`conclusions` 给人看
4. **独立可测**：每个分析器 `analyze(symbol)` → `AnalysisResult`
5. **LLM 聚合有硬规则兜底**：先规则判断再 LLM，不把底线交给黑盒
