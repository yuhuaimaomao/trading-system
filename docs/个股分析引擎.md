# 个股分析引擎设计

> 2026-06-07，基于实际 DB 和代码

---

## 一、定位与边界

个股分析引擎（`stock/`）是独立于实时管线的分析能力层。它是"查一只股票"的工具，被盘前策略、盘中盯盘、盘后复盘、Telegram 交互等各条线 import 使用。

**做：** 多维度分析 → 结构化结论 + 风险标签
**不做：** 交易决策（那是 `trade/decision/` 的事）、实时监控（那是 Watcher 的事）、自动下单

## 三、数据结构

```python
@dataclass
class StockAnalysisRequest:
    symbol: str              # "600519"
    dimensions: list[str]    # ["technical", "news_sentiment", "lhb"]
    params: dict             # {"days": 60, "telegraph_days": 7}

@dataclass
class AnalysisResult:
    dimension: str           # "news_sentiment"
    ok: bool                 # 该维度是否健康
    data: dict               # 结构化数据，给程序消费
    conclusions: list[str]   # 结论短句，给人看
    risk_flags: list[str]    # 风险标签，如"近3日密集利空电报"
    error: str               # 异常信息

@dataclass
class StockAnalysisReport:
    symbol: str
    name: str
    results: list[AnalysisResult]
    aggregated: dict         # {"ok": bool, "total_risks": int, "risk_summary": [...]}
```

`AnalysisResult` 的 `data` 字段是各分析器之间的契约——Watcher/策略线消费 `data` 做程序化判断，CLI/Telegram 消费 `conclusions` 和 `risk_flags` 做展示。

### 4.4 基本面 FundamentalAnalyzer 🆕

**数据源**：`stock_basic`（pe_ttm, pb_ratio, revenue_growth, profit_growth, total_market_cap, circ_market_cap）

Reader 已就绪：`StockReader.get_stock_basic()` 返回这些字段。

**分析逻辑**

| 检查项 | 逻辑 |
|--------|------|
| PE(TTM) | <0 亏损告警 / 0-15 低估 / 15-30 合理 / 30-60 偏高 / >60 高估 |
| PB | <1 破净 / 1-3 合理 / >5 高估 |
| 营收增速 | >20% 高成长 / 0-20% 稳健 / <0 萎缩风险 |
| 利润增速 | >30% 高成长 / 营收增但利润降=成本压力 / 双降=经营恶化 |
| 市值 | >1000 亿大盘 / 100-1000 中盘 / <100 小盘 |

**输出示例**

```
✅ 基本面健康：PE 19.2 合理区间，营收+6.3% 利润+1.5%，流通市值 1.59 万亿
```

```
⚠️ PE -15.3 亏损，营收-8.2% 利润-45% 双降，经营面承压
```

### 4.5 新闻舆情 NewsSentimentAnalyzer 🆕

**数据源**：`cls_telegraph`

**关键字段**：trade_date, level, title, content, stock_tags, ai_sentiment, ai_impact, ai_stocks, ai_sectors, ai_importance, ai_direction

**查询方式**：`stock_tags LIKE '%600519%'` 或 `ai_stocks LIKE '%600519%'`，按 trade_date 倒序取最近 N 天（默认 7 天）。

**需要新建** `StockReader.get_telegraphs(conn, code, days)`。

**分析逻辑**

| 检查项 | 逻辑 |
|--------|------|
| 电报密度 | 近 N 天关联电报数量，>5 条="消息密集期" |
| 情绪分布 | ai_sentiment 利好/利空/中性 的计数，利好占比>60%=偏正面 |
| 重要度 | ai_importance 1-5，>=4 的列出标题 |
| 板块映射 | ai_sectors 提取关联概念板块名称 |
| 时间衰减 | 最近 24h 的权重 > 24-72h > 72h+ |

**输出示例**

```
✅ 近 7 天 3 条相关电报，情绪偏正面（利好 2/中性 1）
  重要：深圳出台跨境支付政策（利好，AI芯片板块）
```

```
⚠️ 近 3 天 6 条相关电报，密集利空（利空 4/中性 2）
  重要：公司收到年报问询函 风险等级 ⭐⭐⭐⭐
```

### 4.7 龙虎榜 LhbAnalyzer 🆕

**数据源**：`lhb_stocks` + `lhb_seats` + `lhb_seat_knowledge`

**lhb_stocks 关键字段**：trade_date, reason, net_inflow, buy_amount, sell_amount, post_1d/2d/5d/10d_change, interpretation

**lhb_seats 关键字段**：seat_type, seat_name, buy/sell/net_amount, is_institution, is_hot_money, seat_tags

**lhb_seat_knowledge 关键字段**：seat_name, style, tags, description

**查询方式**：lhb_stocks 按 stock_code + trade_date 取最近上榜记录；lhb_seats 关联 stock_code + trade_date 取席位明细；lhb_seat_knowledge 按 seat_name 查关联的知名席位风格。

**需要新建** `StockReader.get_lhb(conn, code)`。

**分析逻辑**

| 检查项 | 逻辑 |
|--------|------|
| 上榜信息 | 最近上榜日期、原因、净买入/卖出额 |
| 机构参与 | is_institution=1 的席位数量和净额 |
| 游资参与 | is_hot_money=1 的席位数量和净额 |
| 知名席位 | 匹配 seat_knowledge 输出风格（如"章盟主：短线打板，次日必出"） |
| 上榜后走势 | post_1d/2d/5d/10d_change，判断上榜后是否有持续表现 |
| 买卖力量 | 买盘总额 vs 卖盘总额，净流入=买方强 |

**输出示例**

```
🐉 2026-06-02 龙虎榜（涨幅偏离 7%）
  净买入 +3892 万 | 买盘 1.67 亿 vs 卖盘 1.28 亿
  机构席位 2 家净买入 +4200 万 | 游资 1 家净卖出 -308 万
  知名席位：中信上海分公司（敢死队风格）
  上榜后表现：T+1 +2.1% / T+2 +4.3% / T+5 +1.8%
  解读：机构主导买入，游资小幅出逃
```

```
🐉 最近上榜于 2026-05-15，此后未再上榜
  当日净流出 -1050 万，买方弱
  上榜后 T+1 -3.2%，短期承压
```

## 十、设计原则

1. **分析器只做判断**：调 Reader 拿数据 → 分析 → 返回结论和风险标签。不做 DB 直连、不发消息、不下单
2. **Reader 只做查询**：返回原始 dict/list，不做格式化、不做判断
3. **data vs conclusions 分离**：`data` 给程序消费，`conclusions` 给人看
4. **独立可测**：每个分析器 `analyze(symbol)` → `AnalysisResult`，输入输出确定，不依赖全局状态
5. **新增即注册**：实现 `BaseAnalyzer` + `register()`，StockAnalyzer 自动发现
